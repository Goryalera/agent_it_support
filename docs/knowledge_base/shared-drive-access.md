# Доступ к общим сетевым папкам

Инструкция о том, как подключиться к общим сетевым папкам и запросить к ним права. Полезна тем, кому нужно работать с командными файлами, шаблонами и архивами.

Общие папки лежат на файловом сервере по пути `\\fs.company.ru\share`. Внутри — каталоги по отделам и проектам. У каждого каталога есть владелец ресурса, который подтверждает выдачу прав.

## Пошаговая инструкция

1. Подключитесь к корпоративной сети — по Wi-Fi CompanyCorp в офисе или через VPN удалённо. Без сети сервер недоступен.
2. На Windows откройте «Проводник» и в адресной строке введите `\\fs.company.ru\share`. На macOS — Finder → «Переход → Подключиться к серверу» → `smb://fs.company.ru/share`.
3. При запросе авторизации введите корпоративный логин и пароль.
4. Если нужной папки не видно или она не открывается — значит, у вас нет прав. Откройте ServiceDesk на sd.company.ru и выберите услугу «Доступ к сетевой папке».
5. В заявке укажите полный путь к папке и нужный уровень прав — «Чтение» или «Чтение и запись».
6. Заявка уйдёт на согласование владельцу ресурса. После согласования права настраиваются в течение 1 рабочего дня; о готовности придёт письмо на @company.ru.
7. Для удобства закрепите папку: на Windows «Подключить сетевой диск», на macOS добавьте в избранное Finder.

## Типичные проблемы

- `\\fs.company.ru\share` не открывается → вы не в корпоративной сети; подключитесь к CompanyCorp или VPN.
- Папка видна, но файлы не сохраняются → у вас права только на чтение; запросите «Чтение и запись» отдельной заявкой.
- Не знаете владельца ресурса → его не нужно искать вручную, заявка сама уйдёт нужному согласующему по указанному пути.
- После смены пароля диск перестал подключаться → обновите сохранённые учётные данные на актуальный пароль.
- Папка пропала / переименована → возможно, ресурс перенесли; уточните новый путь у владельца или в поддержке.
